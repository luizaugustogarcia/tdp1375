Proof syntax

The case analysis is performed by extending the configurations that do not have $\frac{11}{8}$-sequences or presenting it, case the configuration has a $\frac{11}{8}$-sequence.

Note that several configurations are equivalent under circular shifts and mirroring. When performing computations, the proof generator only takes one representative of a set of equivalent configurations, called canonical configuration. This configuration is selected as the one with the least associated hash code, among its equivalent ones.

Each generated configuration have its own HTML file, which can be either a file presenting a $\frac{11}{8}$-sequence or a list of all possible extensions of it.

It is important to note that, although each file of the proof presents one or more graphs, the method employed in this work do not rely on them. We have decided to present the graph aiming to facilitate the understanding of the proof for the readers used to the formalisms based on the cycle graph.

Extension files

The extension file starts presenting a cycle graph corresponding to the analysed configuration. Then we present its hash code and a list containing the indices of its open gates. The signature of the configuration is presented. The signature is computed first assigning a label (a integer value) to each cycle in the $\spi$, according to the order of appearance of its symbols in $\epi$. Then, we map each symbol of $\epi$ to the label of the $\epi$ cycle which contains that symbol. In case of oriented cycles, we add to the mapped value, a fractional corresponding to the ordinal position of the symbol in $\epi$. Then, the 3-norm is presented.

Following, the extensions grouped by Type are presented. Each extension starts with the indices (a, b and c) where the new symbols were inserted into the extended configuration. Then, the proof tells if the extension is either BAD or GOOD, highlighted in red and green, respectively. Bad extensions do not have $\frac{11}{8}$-sequences, while the good ones do. This information is followed by the hash code, $3$-norm and signature of the extension. Next, we present a link with the new $\spi$ generated by directly from the insertion of new symbols into the extended configuration (and not its canonical representative). Clicking this link opens a popup window with its associated graph.

Finally, a link to the canonical configuration of the extension allows the reader to visualise either a $\frac{11}{8}$-sequence file, case the extension is good, or another extension file, otherwise.

$\frac{11}{8}$-sequence files

The $\frac{11}{8}$-sequence file first present a graph of the associated configuration. Then, for each transposition $\tau_i$, in the sequence, represented as a $3$-cycle, it shows $\tau_i$, the permutation $\spi{\tau_1}^{-1}\dots{\tau_i}^{-1}$ and the graph corresponding to $\spi{\tau_1}^{-1}\dots{\tau_i}^{-1}$.
